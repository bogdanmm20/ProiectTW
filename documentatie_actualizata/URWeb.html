U R Web

U R Web este o aplicatie web cu scopul de facilita user-ului anumite locatii de o anumita categorie specificata de utilizator.

Tehnologii folosite

PHP

PHP (Php:Hypertext Preprocessor) este un limbaj de programare server-side,de tip script,interpretat. Folosit inițial pentru a produce pagini web dinamice, este folosit pe scară largă în dezvoltarea paginilor și aplicațiilor web. Se folosește în principal înglobat în codul HTML, acesta fiind si modul in care il vom folosi in aplicatia noastra. Pentru a ne ajuta in implementarea aplicatiei ,am ales sa folosim un framework si anume Laravel. Laravel este un framework php disponibil gratuit,in regim open-source ,creeat de Taylor Otwell cu scopul de a servi drept unealta in dezvolatarea aplicatiilor web bazate pe patternul arhitectural MVC(Model-View-Controller).

JavaScript

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor funcționalități în paginile web, codul Javascript din aceste pagini fiind rulat de către browser. Cea mai des întâlnită utilizare a JavaScript este în scriptarea paginilor web. Programatorii web pot îngloba în paginile HTML script-uri pentru diverse activități cum ar fi verificarea datelor introduse de utilizatori sau crearea de meniuri și alte efecte animate. Vom folosi JavaScript pentru ferestre de tip popup,slideshows si pentru a accesa serviciile oferite de platformele pe care le integram. Aceste platforme ne pun la dispozitie diverse API-uri (Application Programming Interface) , pe care le vom folosi pentru a integra continutul deja existent pe platformele mentionate,in aplicatia noastra.

In continuare vom descrie aceste API-uri:

Google Maps API

Am construit pe baza API-urilor de la Google Maps o pagina care afiseaza traseul intre doua puncte. Acest lucru ne va fi folositor cand aplicatia o sa fie implementata, iar un utilizator va putea alege un pin de pe harta si i se va arata traseul pana acolo. Pe langa afisare, am construit o functie care preia functia computeDistanceBetween oferita de Google pentru a determina locatia intre doua pin-uri.

Google Search Places API

Google Search Places este un API pe care o sa il folosim pentru a afisa locatiile pe baza coordonatelor utilizatorului preluate din browser.

Aplicatia noastra prin implementarea Search Places obtine in functie de locatia user-ului toate locatiile de o anumita categorie aleasa de user pe o raza stabilita tot de user.

Google Text Search API

Google Text Search este un API prin care un user tasteaza o anumita locatie in caseta text box si i se va afisa rezultate pe baza locatiilor existente de la Google.

JSON

JSON este un acronim în limba engleză pentru JavaScript Object Notation, și este un format de reprezentare și interschimb de date între aplicații informatice. Este un format text, inteligibil pentru oameni, utilizat pentru reprezentarea obiectelor și a altor structuri de date și este folosit în special pentru a transmite date structurate prin rețea, procesul purtând numele de serializare. JSON este alternativa mai simplă, mai facilă decât limbajul XML. Eleganța formatului JSON provine din faptul că este un subset al limbajului JavaScript (ECMA-262 3rd Edition) , fiind utilizat alături de acest limbaj. Formatul JSON a fost creat de Douglas Crockford și standardizat prin RFC 4627. Tipul de media pe care trebuie să îl transmită un document JSON este application/json. Extensia fișierelor JSON este .json.

Arhitectura

Aplicatia va fi proiectata pe arhitectura MVC. In functie de actiunile selectate de utilizator , browserul va naviga catre o alta pagina ( multipage ).

  1. Homepage

    Este prima pagina pe care un utilizator o intalneste.

    Aceasta pagina va contine harta care nu are nicio functionalitate, iar in partea de sus este un buton de Log In care il va duce pe user la pagina de logare.

  2. Sign up

    Pentru inregistrare , userul va introduce numele si parola cu care ar dori sa se logheze in aplicatie . Un script va verifica daca ambele campuri au fost completate corect . Deasemenea , scriptul va verifica daca mai exista si alti utilizatori cu acelasi nume ( acesta trebuie sa fie unic ) .

  3. Login

    Dupa etapa de inregistrare , userul se va loga , iar acelasi script va verifica validarea datelor . Odata autentificat , userul poate selecta din meniu ce optiune doreste . Are la indemana urmatoarele optiuni: Add location, Favourite list, Search, Choose category.

  4. Add location

    Deasemenea , un utilizator va fi capabil sa adauge o locatie la baza de date . Pagina va afisa un formular in care un utilizator va adauga date despre locatie. Dupa ce s-au adaugat datele, locatia noua va fi inserata in baza de date.

  5. Favourite locations

    Aceasta pagina are ca functionalitate ca un utilizator sa adauge o locatie preferata. Dupa ce este adaugata in baza de date, aceste locatii se vor afisa pe o pagina ca un utilizator sa le aibe la indemana.

  6. Search

    Un utilizator are la indemana o harta cu un text search box in care el poate sa caute o locatie si pe baza API-ului, i se vor afisa locatii pe baza input-ului si user-ul va selecta una ca sa o caute pe harta.

  7. Choose category

    Aceasta pagina ofera functionalitatea de a afisa obiective de interes pe harta pe baza locatiei utilizatorului. Un utilizator daca vrea sa selecteze o categorie va apasa pe Choose category, astfel un meniu va aparea cu o lista de butoane care au denumire categoria pe care vrea sa o selecteze utilizatorul. Pe aceeasi pagina este un label care un utilizator va putea sa insereze raza pe care el vrea sa o stabileasca. Cand va apasa pe butonul cu categorie, va fi redirectionat catre o harta si i se vor afisa pin-uri pe harta cu numele locatiei.

    8. Detalii de implementare

    Modelarea datelor

    Pentru stocarea datelor vom folosi serveru de baze de date Oracle .

    Baza de date va contine tabelele :

    • Tabela Users - va contine campurile : name , password si id_user ( care va fi unic pentru fiecare utilizator ).
    • Tabela Preferences va contine id_user si 10 campuri pref_location1 , pref_location2 .. pref_location10 care vor memora locurile preferate ( id_location ) adaugate de utilizator ( de exemplu , pizzeria Mamma Mia , restauratul Oscar , locuinta unui prieten , etc ) .
    • Tabela Locations va contine toate locatiile inregintrate si va avea urmatoarele campurile : id_location , name , category , adress , nr_telefon , descriere .
